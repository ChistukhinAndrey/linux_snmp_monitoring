## 1. Цель работы

Целью работы является разработка системы мониторинга Linux-сервера с использованием протокола SNMP, системы мониторинга Zabbix и системы визуализации Grafana.

Задачи работы:

- развернуть сервер Zabbix в Docker-контейнере
- установить систему визуализации Grafana
- настроить SNMP-агент на целевом сервере Debian
- настроить мониторинг метрик в Zabbix
- создать dashboard для отображения метрик в Grafana

---

## 2. Используемые технологии

В работе использовались следующие технологии:

- Debian Linux
- Docker
- Zabbix
- Grafana
- SNMP (Simple Network Management Protocol)

---

## 3. Архитектура системы

Система мониторинга состоит из трех основных компонентов:

1. Целевой сервер Debian с установленным SNMP-агентом
2. Сервер мониторинга Zabbix
3. Система визуализации Grafana

Принцип работы:

1. SNMP-агент собирает системные метрики.
2. Zabbix опрашивает SNMP-агент.
3. Метрики сохраняются в базе данных.
4. Grafana отображает данные в виде графиков.

---

## 4. Развертывание системы мониторинга

Для развертывания использовалась технология Docker.

Был создан файл docker-compose.yml, содержащий контейнеры:

- Zabbix Server
- Zabbix Web Interface
- MySQL database
- Grafana

Запуск системы выполняется командой:

docker compose up -d

После запуска сервисы доступны:

Zabbix Web Interface  
http://localhost:8080

Grafana  
http://localhost:3000

---

## 5. Настройка SNMP

На целевом сервере Debian установлен SNMP-агент.

Установка:

sudo apt update  
sudo apt install snmp snmpd

Конфигурация SNMP выполнена в файле:

/etc/snmp/snmpd.conf

Пример конфигурации:

rocommunity public 192.168.1.10

Эта настройка разрешает серверу Zabbix выполнять SNMP-запросы.

После изменения конфигурации сервис был перезапущен:

sudo systemctl restart snmpd

---

## 6. Настройка хоста в Zabbix

В интерфейсе Zabbix был добавлен новый хост.

Параметры:

Host name: debian-server

Interface type: SNMP  
Port: 161  
SNMP version: v2  
Community: public

---

## 7. Настроенные метрики

В системе мониторинга были настроены следующие метрики:

1. Загрузка процессора (CPU Load)
2. Использование оперативной памяти (RAM usage)
3. Использование дискового пространства (Disk usage)
4. Сетевой трафик (Network traffic)

Эти метрики собираются через SNMP и отображаются в системе мониторинга.

---

## 8. Визуализация данных в Grafana

Grafana была подключена к Zabbix с помощью плагина Zabbix datasource.

В Grafana был создан Dashboard, отображающий:

- загрузку CPU
- использование RAM
- использование диска
- сетевой трафик

Графики обновляются в реальном времени.
